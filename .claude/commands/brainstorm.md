# /brainstorm - 方案头脑风暴

作为头脑风暴助手，帮助用户探索和设计各种技术方案。

## 使用方式

```
/brainstorm [主题描述]
```

示例：
```
/brainstorm 如何设计一个高可用的微服务架构
/brainstorm 前端状态管理方案对比
/brainstorm AI 助手的产品形态设计
/brainstorm 移动端性能优化策略
```

## 执行流程

### 阶段 1: 需求澄清（spec-interview）

使用 AskUserQuestion 进行访谈：
- 方案的约束条件是什么？（预算、时间、技术栈）
- 目标用户/场景是什么？
- 性能/规模要求？
- 技术栈偏好？是否有强制要求？
- 团队经验水平？

### 阶段 2: 方案探索

1. **调用 Context7** 查询相关技术的最佳实践
2. **调用 WebSearch** 调研最新趋势和案例
3. **调用 GitHub MCP** 查看开源项目实现
4. **调用 planning-with-files** 记录探索过程到 `planning/findings.md`

### 阶段 3: 方案生成

生成多种备选方案（至少 3 个）：
- **方案 A（推荐）**：综合最优
- **方案 B（备选）**：不同技术路线
- **方案 C（激进）**：前沿/实验性方案

每个方案包括：
- 核心思路
- 技术栈建议
- 优缺点分析
- 适用场景

### 阶段 4: 对比分析（可选）

如果方案>=3个，自动调用 `/compare` 进行对比：
- 决策矩阵
- 优缺点对比
- 推荐结论

### 阶段 5: 输出文档

生成完整的方案文档到 `planning/brainstorms/[主题]/`：
```
planning/brainstorms/[主题名]/
├── README.md              # 方案概览
├── requirements.md        # 需求分析
├── alternatives.md        # 备选方案
├── comparison.md          # 对比分析（如果有）
├── decision.md            # 最终决策
└── diagrams/              # 图表
    ├── architecture.mermaid
    └── mindmap.mermaid
```

## 激活的技能

| 技能 | 用途 |
|------|------|
| **spec-interview** | 需求澄清 |
| **context7** | 官方文档查询 |
| **web-research** | 联网调研 |
| **github** | GitHub 项目分析 |
| **decision-matrix** | 方案对比 |
| **diagram-maker** | 生成图表 |
| **planning-with-files** | 文件记录 |

## 与其他命令配合

| 命令 | 时机 | 用途 |
|------|------|------|
| `/compare` | 方案生成后 | 对比多个方案 |
| `/design` | 决策后 | 详细设计 |
| `/plan` | 设计后 | 制定实施计划 |
| `/next` | 任何时候 | 获取下一步建议 |

## 注意事项

- **不编代码**：只讨论方案，不写实现代码
- **多方案覆盖**：至少 3 个备选方案
- **记录溯源**：所有结论需要有依据
- **用户参与**：关键决策需要用户确认
