# /plan - 方案规划

> 快速创建和管理方案规划

## 使用场景

当用户需要：
- 创建一个新的方案计划
- 查看或编辑现有的任务计划
- 调整计划中的优先级和里程碑

## 触发方式

```
/plan [主题]
```

## 操作流程

### 1. 规划创建

如果用户没有指定主题或需要创建新计划：

1. 询问用户方案主题
2. 使用 `planning-with-files` 技能创建 `planning/task_plan.md`
3. 包含以下部分：
   - 方案概述
   - 目标
   - 关键里程碑
   - 待解决问题

### 2. 规划查看

如果用户想要查看现有计划：

1. 检查 `planning/task_plan.md` 是否存在
2. 显示当前计划内容
3. 标记已完成/未完成项

### 3. 规划调整

如果用户需要修改计划：

1. 读取当前计划
2. 询问需要修改的部分
3. 更新文件内容

## 输出格式

### 计划文件结构

```markdown
# 方案任务计划

## 概述
[方案简介]

## 目标
- [ ] 目标 1
- [ ] 目标 2

## 里程碑
| 阶段 | 描述 | 状态 |
|------|------|------|
| 阶段 1 | 初步调研 | pending |
| 阶段 2 | 方案设计 | pending |
| 阶段 3 | 详细设计 | pending |

## 待解决问题
- [ ] 问题 1
- [ ] 问题 2

## 进度
- [ ] 阶段 1: 初步调研
- [ ] 阶段 2: 方案设计
- [ ] 阶段 3: 详细设计

---
创建时间: {date}
```

## 注意事项

- 遵循 `planning-with-files` 技能规范
- 所有规划写入文件而非仅在内存
- 保持计划文件与实际进度一致
