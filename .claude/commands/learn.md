## 中文说明
- 功能：提取可复用模式
- 使用：`/learn`

## 英文原文

# /learn - Extract Reusable Patterns

分析当前会话并提取任何值得保存为技能的模式。

## 触发

在会话中解决非平凡问题时随时运行 `/learn`。

## 提取什么

寻找：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 什么修复了它？
   - 这对于类似错误可复用吗？

2. **调试技术**
   - 非显而易见的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **变通方案**
   - 库的特性
   - API 限制
   - 版本特定的修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/` 创建技能文件：

```markdown
# [描述性模式名称]

**提取日期**: [日期]
**上下文**: [适用场景的简要描述]

## 问题
[解决什么问题 - 具体]

## 解决方案
[模式/技术/变通方案]

## 示例
[如有适用，代码示例]

## 何时使用
[触发条件 - 什么应该激活此技能]
```

## 过程

1. 回顾会话中可提取的模式
2. 确定最有价值/可重用的洞察
3. 起草技能文件
4. 保存前询问用户确认
5. 保存到 `~/.claude/skills/learned/`

## 注意

- 不要提取简单的修复（拼写错误、简单语法错误）
- 不要提取一次性问题（特定 API 中断等）
- 专注于未来节省时间的模式
- 保持技能专注 - 每个技能一个模式
