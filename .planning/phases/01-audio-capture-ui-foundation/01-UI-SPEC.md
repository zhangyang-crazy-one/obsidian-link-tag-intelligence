> Visual and interaction contract for Phase 1: Audio Capture & UI Foundation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Obsidian plugin -- shadcn not applicable) |
| Preset | not applicable |
| Component library | Obsidian Plugin API (`PluginSettingTab`, native HTML elements) |
| Icon library | Obsidian built-in icons (lucide-based) |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background-primary)` / `var(--background-secondary)` (Obsidian theme) | All background surfaces, settings panel background |
| Secondary (30%) | `var(--background-secondary-alt)` (Obsidian theme) | Settings section cards, toolbar background |
| Accent (10%) | `var(--interactive-accent)` (Obsidian theme) | Recording-active state indicator: button icon color during recording, pulsing border color |
| Destructive | `#e93147` | Error state: button background tint when error occurs, error icon color. **Destructive actions only.** |
| VU green | `#4caf50` | VU bars 1-3 during normal speech levels (<= -18 dBFS) |
| VU yellow | `#ff9800` | VU bars 4 during elevated levels (-18 to -6 dBFS) |
| VU red | `#f44336` | VU bar 5 during peak/clipping (> -6 dBFS) |
| VU bar idle | `var(--text-faint)` at 30% opacity | Empty/unfilled VU bars |

**Accent reserved for:** Recording-active state indication only -- toolbar button icon color shift, pulsing border animation, and the filled portion of VU meter bars during recording. Never used for static UI chrome, settings controls, or idle state.

**VU meter per-bar thresholds (5 bars, reference: dBFS):**
- Bar 1: fills when level > -50 dBFS (green)
- Bar 2: fills when level > -36 dBFS (green)
- Bar 3: fills when level > -24 dBFS (green)
- Bar 4: fills when level > -18 dBFS (yellow)
- Bar 5: fills when level > -6 dBFS (red)

## Interaction & Animation Contract

### Recording Button States

| State | Visual | CSS class |
|-------|--------|-----------|
| `idle` | Default mic icon, no animation, transparent background | `lti-speech-record-btn` (no modifier) |
| `initializing` | Mic icon with spinning loader indicator (Obsidian `--icon-loader-2` or equivalent), 1s rotation animation | `lti-speech-record-btn is-initializing` |
| `recording` | Mic icon color shifted to accent, border pulsing (breathing effect), VU meter visible | `lti-speech-record-btn is-recording` |
| `processing` | Checkmark or processing icon, subtle fade pulse, VU meter hidden | `lti-speech-record-btn is-processing` |
| `error` | Button background tinted red at 15% opacity, error icon (alert-triangle) overlaid, Notice displayed | `lti-speech-record-btn is-error` |

### Pulsing Animation (Recording)

```css
@keyframes lti-speech-pulse {
  0%, 100% { box-shadow: 0 0 0 0 var(--interactive-accent-hover, rgba(128, 128, 255, 0.45)); }
  50% { box-shadow: 0 0 0 4px rgba(128, 128, 255, 0.15); }
}
```

- Duration: 1.5s per cycle
- Easing: `ease-in-out`
- Infinite loop while in `recording` state
- Applied via `box-shadow` to avoid layout shift
- Transparent background -- no fill color change on the button itself

### VU Meter Behavior

- 5 vertical bars, 4px wide each, 2px gap, 18px tall (max)
- Bars fill from left to right as dBFS increases
- Each bar transitions from empty (30% opacity text-faint) to its threshold color
- Numeric dB readout displayed to the right of bars in 12px font
- Entire VU meter component is hidden (display: none) when not in `recording` state
- Update rate: ~60ms (tied to AudioWorklet analyser output, throttled to ~16fps for visual stability)

### Error State Transitions

- Entry: Button class switches to `is-error` immediately on error detection
- Obsidian `Notice` fires with specific error message (auto-dismiss after 8 seconds)
- Recovery: User must click the error-state button to confirm (dismisses error, returns to `idle`)
- Mic unplugged during recording: auto-detect via `navigator.mediaDevices.ondevicechange`, immediately stop recording, transition to `idle` with Notice

### Destructive Actions

No destructive actions in Phase 1. The toggle recording command is reversible and does not delete data. Error recovery is a single-click confirm (click error-state button).
