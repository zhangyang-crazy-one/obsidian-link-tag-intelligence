<objective>
Create the sherpa-onnx Web Worker engine and integrate it with the SpeechRecorder's audio pipeline. Audio chunks flow from microphone capture through ArrayBuffer transfer to a dedicated Web Worker running the sherpa-onnx WASM ASR engine. The Worker is created lazily on first toggle (not during plugin load), handles streaming decode with endpoint detection, and communicates results back to the main thread via postMessage protocol.

Purpose: Foundation for Phase 02 — all subsequent plans depend on the Worker protocol for text output and model path resolution.
Output: Working Web Worker with sherpa-onnx WASM that accepts audio chunks, performs streaming decode, and emits sentence-final text results via postMessage. SpeechRecorder extended with Worker lifecycle management.

**Step 4: Create src/speech-worker.ts (Web Worker entry point)**
The Worker file is a standalone CJS module that uses `require()` to load sherpa-onnx from node_modules at runtime (Electron's Node.js integration). It must NOT import from other src/ files — all logic is self-contained.

Key design decisions per CONTEXT.md:
- D-03: sherpa-onnx via WASM npm package (require('sherpa-onnx/...'))
- D-06: 800ms silence endpoint via `rule1MinTrailingSilence: 0.8`
- D-11: Use sherpa-onnx built-in VAD (enableEndpoint: 1)

Worker message protocol:
- Incoming: `{type:'init', modelDir: string, language: 'zh'|'en', vadSensitivity: number}` → load WASM, create recognizer + stream → post `{type:'ready'}`
- Incoming: `{type:'audio', buffer: ArrayBuffer}` → acceptWaveform → decode loop → post `{type:'result', text: string, isEndpoint: boolean}`
- Incoming: `{type:'reset'}` → recognizer.reset(stream)
- Incoming: `{type:'destroy'}` → stream.free(), recognizer.free() → post `{type:'destroyed'}`

**Add waitForWorkerReady helper:**
```typescript
private waitForWorkerReady(timeoutMs: number): Promise<void> {
  return new Promise((resolve, reject) => {
    const timeout = setTimeout(() => {
      reject(new Error('ASR Worker init timed out after ' + timeoutMs + 'ms'));
    }, timeoutMs);
    const checkInterval = setInterval(() => {
      if (this.workerReady) {
        clearTimeout(timeout);
        clearInterval(checkInterval);
        resolve();
      }
    }, 100);
  });
}
```

| main thread → Web Worker | Audio ArrayBuffer data crosses boundary via postMessage transfer; Worker code is self-contained and auditable |
| file system → WASM | Model files (ONNX + tokens.txt) read by sherpa-onnx WASM; path traversal risk if modelDir is user-controlled |
| HuggingFace CDN → plugin | Model download over HTTPS (handled in Plan 03); integrity verified via SHA256 |
| microphone → AudioContext | Raw audio captured by Web Audio API; never written to disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | speech-worker.ts (model file loading) | mitigate | Validate modelDir against expected plugin directory pattern; reject paths containing `..` segments |
| T-02-02 | Tampering | speech-worker.ts (audio data) | accept | Audio data is ephemeral Float32Array — transferred to Worker, processed, garbage collected. No persistence. Low-value target. |
| T-02-03 | Information Disclosure | speech-worker.ts (postMessage output) | accept | Worker only sends text strings; no audio data or model internals leak to main thread. Self-contained code is auditable. |
| T-02-04 | Denial of Service | speech-recorder.ts (Worker lifecycle) | mitigate | Worker init timeout (10s) prevents indefinite hang; Worker terminated on plugin unload; `ArrayBuffer` chunk size limited by AudioWorklet buffer (128 samples at 16kHz = 8ms) |

| T-02-06 | Elevation of Privilege | speech-worker.ts (require() in Worker) | accept | Electron's nodeIntegrationInWorker is a platform feature; Worker code is bundled at build time, not loaded from user data. No user-controlled Worker code. |
