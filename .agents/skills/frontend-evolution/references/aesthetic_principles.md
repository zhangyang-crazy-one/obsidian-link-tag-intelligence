# 审美原则 - 空间、层次、呼吸感

## 目录

- 空间关系原则
- 视觉层次原则
- 呼吸感原则
- 一致性原则
- 实战检验清单
- 设计决策树

> 核心观点：好的设计让用户感到"舒适"，这种舒适感来自合理的空间利用、清晰的层次和适度的呼吸。

## 1. 空间关系原则

### 1.1 按钮控件与文字

**问题**：按钮内的文字位置不正确会破坏视觉平衡。

**正确做法**：
- 文字在按钮内垂直居中
- 按钮高度 ≥ 32px（触控友好）
- 文字与按钮边缘间距 ≥ 8px
- 按钮宽度根据内容自适应，或固定常见尺寸

**示例**：
```
✗ 错误：文字紧贴按钮顶部
┌─────────┐
│ 保存   │  ← y 位置错误，偏上
└─────────┘

✓ 正确：文字垂直居中
┌─────────┐
│  保存   │  ← y 位置正确，居中
└─────────┘
```

### 1.2 表单字段对齐

**核心原则**：同类元素对齐，而非视觉对称。

**问题**：不同长度标签导致输入框起始位置不一致。

```
✗ 错误：标签长度不同导致输入框不对齐
Provider  [____________]
Model     [____________]
API Key   [____________]

✓ 正确：固定标签宽度，输入框统一起始
Provider   [____________]
Model      [____________]
API Key    [____________]
```

**解决方案**：
- 固定标签区域宽度（如 100px）
- 输入框统一从同一 x 位置开始
- 或使用左对齐，标签右边缘对齐

### 1.3 图标与文字

**水平排列**：
- 图标与文字间距 8-12px
- 图标与文字垂直居中对齐
- 图标尺寸与文字大小协调（16px 图标 ↔ 14px 文字）

**垂直排列**：
- 图标在上的情况：图标与下方文字间距 4-8px
- 图标在左的情况：图标与左侧内容间距 8-12px

## 2. 视觉层次原则

### 2.1 信息优先级

| 级别 | 特征 | 场景 |
|------|------|------|
| 主信息 | 大字体(18-24px)、高对比度(#c9d1d9)、粗体 | 标题、重要数字 |
| 次信息 | 中等字体(14-16px)、中等对比度(#8b949e) | 正文、描述 |
| 辅助信息 | 小字体(11-12px)、低对比度(#6e7681) | 标签、时间、状态 |

### 2.2 交互状态

**状态层级**：
```
默认 → Hover → Active → Focus
低    中      高       中(取决于)
```

**设计规范**：
- Hover：背景提亮 10-20%，或添加边框
- Active：背景变暗 10-20%，或添加内阴影
- Focus：明显的轮廓环（如 2px #58a6ff）

### 2.3 面板层次

**背景层次**：
```
最深层：页面背景 (#0d1117)
中间层：卡片/面板 (#161b22, #1a1a2e)
表层：弹出层/模态框 (#1a1a2e + 圆角)
```

**边框使用**：
- 分隔用：1px #30363d
- 强调用：1px #58a6ff
- 危险用：1px #da3633

## 3. 呼吸感原则

### 3.1 元素间距系统

**基础间距单位**：4px

| 名称 | 值 | 用途 |
|------|-----|------|
| xs | 4px | 紧密相关元素内部 |
| sm | 8px | 相关元素之间 |
| md | 12px | 一般间距 |
| lg | 16px | 组件之间 |
| xl | 24px | 区块之间 |
| 2xl | 32px | 大区块之间 |
| 3xl | 48px | 页面级间距 |

### 3.2 呼吸感检验

**检验问题**：
1. 用户视线能否自然地从 A 点移动到 B 点？
2. 元素之间是否有"喘息"的空间？
3. 密集信息是否被适当的空白分隔？

**常见问题**：
```
✗ 拥挤：元素紧贴边缘
┌────────────────────┐
│标签[输入框]标签[输入框]│
│标签[输入框]标签[输入框]│
└────────────────────┘

✓ 呼吸：适当内边距和间距
┌────────────────────┐
│                      │
│  标签   [输入框]     │
│  标签   [输入框]     │
│                      │
└────────────────────┘
```

### 3.3 面板边缘

**面板边缘内边距**：
- 小型面板：16px
- 中型面板：24px
- 大型面板：32px

**模态框特殊规则**：
- 模态框与背景间距：至少 48px
- 内容与模态框边缘：至少 24px

## 4. 一致性原则

### 4.1 尺寸一致性

**同类元素使用相同尺寸**：
```
按钮高度：32px / 36px / 44px（仅三种规格）
图标尺寸：16px / 20px / 24px / 32px
输入框高度：32px / 40px / 48px
```

**间距倍数**：
- 使用 4 的倍数：4, 8, 12, 16, 24, 32, 48
- 避免使用 7, 13, 17 等奇怪数值

### 4.2 圆角一致性

**圆角系统**：
```
无圆角：0px      - 分隔线、边框
小圆角：4px      - 标签、小按钮
中圆角：6-8px   - 按钮、输入框
大圆角：12-16px - 卡片、面板
特大圆角：20px+ - 模态框（谨慎使用）
```

### 4.3 颜色一致性

**同一语义使用同一颜色**：
- 成功：#3fb950
- 错误：#da3633
- 警告：#d29922
- 信息：#58a6ff

**同一状态使用同一模式**：
- 选中态：背景 #238636
- 未选中态：背景 #21262d

## 5. 实战检验清单

### 设计完成后自检

- [ ] 按钮文字是否垂直居中？
- [ ] 同类元素是否左对齐？
- [ ] 输入框起始位置是否统一？
- [ ] 元素间距是否符合间距系统？
- [ ] 面板边缘是否有足够内边距？
- [ ] 相似元素尺寸是否一致？
- [ ] 交互状态是否可见？
- [ ] 整体是否有"呼吸感"？

### 用户反馈对齐

当用户说"感觉不对"时，可能的问题：
1. 空间关系：元素之间太挤或太散
2. 层次不清：主次信息混淆
3. 呼吸感差：整体太紧凑或太松散
4. 一致性差：相似元素不统一

## 6. 设计决策树

```
用户反馈"不美观"
├── 拥挤感？
│   ├── 减少元素间距
│   ├── 增加面板内边距
│   └── 移除不必要的元素
├── 层次混乱？
│   ├── 放大主信息
│   ├── 缩小次信息
│   └── 增加视觉分隔
├── 不对齐？
│   ├── 统一标签宽度
│   ├── 统一输入框起始
│   └── 使用 grid 布局
└── 太平淡？
    ├── 增加强调色
    ├── 添加圆角
    └── 增加阴影/深度
```
